# Centralized College Information Management System

## 📌 Project Overview

This project is a centralized web-based platform designed to improve communication and information sharing within an educational institution. It provides a single point of access for students, faculty, club members, and administrators to manage and view college-related updates efficiently.
The platform consolidates:

College circulars
Club events
Faculty webinars
Notifications and updates
By replacing scattered communication channels like notice boards, emails, and messaging apps, this system ensures clarity, accessibility, and real-time updates.

## 🎯 Problem Statement

In many colleges, important announcements and event details are shared across multiple platforms, leading to:
-Missed information
-Lack of transparency
-No centralized tracking
-Poor user experience
-There is a need for a secure, scalable, and role-based system that provides all academic and co-curricular updates in one place.

## 💡 Proposed Solution

The proposed solution is a centralized web platform that offers:
-Role-based access control
-Real-time data updates
-Secure authentication
-Easy content management
-Each user interacts only with features relevant to their role, improving usability and system security.

## 👥 User Roles

Student – View circulars, events, webinars, and register for events
Faculty – Post circulars and manage webinars
Club Member – Create and manage club events
Admin – Manage users, content, and analytics

## ✨ Key Features

-Student Module
-View college circulars
-View and register for club events
-Access webinar details
-Receive notifications
-Faculty Module
-Upload and manage circulars
-Create and manage webinars
-Club Member Module
-Post club events

-Admin Module

## 🛠 Google Technologies Used

Firebase Authentication – Secure login using Email/Password and Google Sign-In
Firebase Firestore – Real-time NoSQL database for storing users, events, and circulars
Firebase Hosting – Hosting and deployment of the web application
Google Calendar (Optional) – Event and webinar scheduling
Google Cloud Platform (GCP) – Backend cloud infrastructure
Anti-Gravity Platform – Used for rapid development, faster prototyping, and cloud integration support

## 🔐 Security Features
Role-based access control
Encrypted authentication
Secure cloud storage
Controlled content access

## 🏗 System Architecture (High-Level)
User accesses the web application
Authentication is handled by Firebase Authentication
Data is stored and retrieved from Firestore
Files are managed via Firebase Storage
Application is hosted using Firebase Hosting
